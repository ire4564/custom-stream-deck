## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
커스텀 가능한 가상 스트림덱

### 배경
오늘날 macOS 사용자들은 콘텐츠 제작, 개발, 사무 업무 등 다양한 분야에서 반복적이고 빈번한 작업을 수행합니다. 이러한 작업을 더 효율적으로 처리하기 위해 물리 스트림덱, 키보드 단축키, 제스처 등의 다양한 방법을 활용하지만, 이들 방법은 직관성, 개인화, 시각적 피드백 등에서 한계를 가질 수 있습니다. 본 서비스는 이러한 문제점을 해결하고 사용자에게 더욱 몰입감 있고 즐거운 작업 환경을 제공하기 위해 기획되었습니다.

### 사용자 문제
macOS 사용자들이 자주 수행하는 작업 실행 속도가 느리고, 기존의 작업 실행 방식(물리 스트림덱, 키보드 단축키, 제스처 등)은 직관성이 부족하거나 개인화에 한계가 있어 효율성과 만족도가 떨어집니다.

### 해결 방식
macOS 데스크톱 위에 가볍게 띄워두는 '시각형 액션 런처' 컨셉의 가상 스트림덱 앱을 제공하여, 사용자가 고도로 커스터마이징 가능한 버튼들을 통해 자주 사용하는 작업을 빠르고 직관적으로 실행할 수 있도록 합니다.

### 차별점
단순한 기능 실행을 넘어 아이콘, GIF, 라벨, 배경색, 호버 애니메이션, 액티브/진행 상태 등으로 각 버튼을 자유롭게 꾸밀 수 있는 '무료 커스터마이징 및 개인화' 기능을 제공하여, 사용자의 개성을 극대화하고 '상태가 살아있는 UI' 경험을 제공함으로써 기존 물리 스트림덱이나 다른 소프트웨어 런처와 차별화됩니다.

### 타겟 사용자
콘텐츠 크리에이터: 영상 편집, 이미지 보정, 스트리밍 중 자주 사용하는 앱 실행, 특정 기능 호출, 스크립트 실행 등을 신속하게 처리하려는 사용자. 개발자: 개발 환경 설정, 빌드, 테스트 스크립트 실행, 터미널 명령 입력 등을 버튼 하나로 해결하고 싶은 사용자. 사무직 종사자: 특정 문서 열기, 웹사이트 접속, 회의 도구 실행, 매크로 작업 등을 효율적으로 수행하려는 사용자. macOS 환경에서 반복적인 작업이 많고 효율성 및 개인화를 중시하는 모든 사용자.

### 사용 시나리오
사용자는 영상 편집 중 자주 사용하는 효과 플러그인 실행, 특정 폴더 열기, 또는 스크립트 실행을 위해 가상 스트림덱의 버튼을 누릅니다. 또한, 콘텐츠 제작 워크플로우를 위해 필요한 애플리케이션을 빠르게 실행하거나, 개발자가 복잡한 커스텀 스크립트를 버튼 하나로 실행하여 작업 시간을 단축하는 등 반복적인 작업을 처리해야 하는 모든 상황에서 가상 스트림덱을 활용하여 작업 효율성을 극대화하고 개인화된 환경을 즐길 것입니다.

### 제품 목표
macOS 사용자가 자주 사용하는 작업을 시각적이고 직관적인 인터페이스를 통해 빠르고 효율적으로 실행하며, 사용자의 개성을 반영한 고도로 개인화된 작업 환경을 구축할 수 있도록 지원합니다.

### 핵심 KPI
사용자 만족도 및 리뷰 점수(평점 4.5 이상), 주간 활성 사용자(WAU) 수 1만명 달성, 월 평균 버튼 클릭 횟수 100회 이상, 앱 내 프로필 생성 및 공유 횟수 증가.

### 리스크/이슈
물리 스트림덱의 하드웨어적 직관성과 물리적 피드백을 소프트웨어로 완전히 대체하기 어려울 수 있으며, macOS 시스템 통합 및 안정성 문제 발생 가능성이 있습니다. 또한, 사용자 커스터마이징의 복잡성으로 인해 초기 진입 장벽이 높아질 위험도 존재합니다.

### 사용자 역할

## 1. 가상 스트림덱 레이아웃 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 2x2, 3x3, 4x2, 5x3 등 프리셋 레이아웃 또는 자유 배치를 통해 자신만의 덱을 구성할 수 있어야 합니다. 미니/컴팩트/노멀/라지 크기 및 세로형/가로형/플로팅 아일랜드 형태 등 다양한 레이아웃을 지원해야 합니다.

**수용 기준:**
```
1. ○ 사용자가 2x2, 3x3, 4x2, 5x3 등 기본 레이아웃 중 하나를 선택하여 스트림덱을 생성할 수 있어야 합니다.
2. ○ 사용자가 스트림덱의 크기를 mini, compact, normal, large 중 선택할 수 있어야 합니다.
3. ○ 사용자가 스트림덱의 표시 형태를 세로형, 가로형, 플로팅 아일랜드 중 선택할 수 있어야 합니다.
4. ○ 생성된 스트림덱은 macOS 데스크톱 위에 항상 가볍게 띄워져 표시되어야 합니다.
```


### 1.1 프리셋 레이아웃 선택
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

2x2, 3x3, 4x2, 5x3 등 프리셋 레이아웃을 선택해 덱을 빠르게 만들 수 있어야 합니다.


### 1.2 자유 배치 편집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

버튼을 자유롭게 배치하거나 그리드 스냅을 활용해 배치를 조정할 수 있어야 합니다.


### 1.3 덱 크기 설정
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

mini/compact/normal/large 등 덱 크기를 선택해 화면 점유를 조절할 수 있어야 합니다.


### 1.4 덱 표시 형태 설정

## 2. 버튼 액션 등록 및 실행
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 각 버튼에 macOS 애플리케이션 실행, 파일/폴더 열기, 커스텀 스크립트/자동화 실행 등의 액션을 할당하고, 해당 버튼 클릭 시 지정된 액션이 지연 없이 즉시 실행되어야 합니다.

**수용 기준:**
```
1. ○ 사용자가 버튼에 macOS 애플리케이션 실행 액션을 등록할 수 있어야 합니다.
2. ○ 사용자가 버튼에 특정 파일 또는 폴더 열기 액션을 등록할 수 있어야 합니다.
3. ○ 사용자가 버튼에 쉘 스크립트 또는 자동화 스크립트 실행 액션을 등록할 수 있어야 합니다.
4. ○ 등록된 버튼 클릭 시 해당 액션이 지연 없이 실행되어야 합니다.
```

설치된 앱 목록을 검색하거나 최근 사용 앱에서 선택해 버튼에 연결할 수 있어야 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS


#### 2.1.2 실행 옵션(새 창/포커스)

앱 실행 시 새로 실행하거나, 이미 실행 중이면 포커스만 주는 등 실행 옵션을 제공해야 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS


### 2.2 파일/폴더 열기 액션
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

버튼에 특정 파일 또는 폴더 열기 액션을 할당하고 실행할 수 있어야 합니다.


#### 2.2.1 경로 선택 및 유효성 검사

파일/폴더 경로를 선택하고, 존재하지 않는 경로일 때 경고 및 수정 유도를 제공해야 합니다.

### 2.3 스크립트/자동화 실행 액션
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

## 3. 버튼 시각적 커스터마이징
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 각 버튼의 아이콘, GIF, 라벨 텍스트, 배경색 등을 자유롭게 설정하여 고도로 개인화된 UI를 구성할 수 있어야 합니다.

**수용 기준:**
```
1. ○ 사용자가 버튼에 표시될 아이콘을 직접 업로드하거나 내장된 아이콘 라이브러리에서 선택할 수 있어야 합니다.
2. ○ 사용자가 버튼에 GIF 애니메이션을 적용할 수 있어야 합니다.
3. ○ 사용자가 버튼의 라벨 텍스트를 입력하고 폰트, 크기, 색상을 설정할 수 있어야 합니다.
4. ○ 사용자가 버튼의 배경색을 팔레트에서 선택하거나 HEX 코드로 지정할 수 있어야 합니다.
```

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS

아이콘 변경 시 즉시 미리보기가 반영되어야 하며, 되돌리기(초기화)로 기본 상태로 복원할 수 있어야 합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: macOS
